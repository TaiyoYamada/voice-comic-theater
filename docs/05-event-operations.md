# 05. 本番当日の運用手順

10人の子どもが iPad で同時に作品を作るイベントを想定した手順です。

---

## 役割分担（目安）

- **進行担当**: 全体のステップを声かけ。
- **TA / 先生**: 接続トラブル対応（管理画面 `/admin` を使う）。

---

## 事前準備（前日まで）

1. **GAS + Sheets** をデプロイ（[04-gas-sheets.md](04-gas-sheets.md)）。`GAS_URL` を確定。
2. **フロントを固定URLにホスティング**（Vercel など）。`VITE_GAS_URL` を本番値でビルド。
   - 子どもは当日この固定URLを開くだけ（QRは予備）。
3. **パネル画像を差し替え**（`frontend/public/panels/` + `manifest.json`）。
4. **Colab notebook** を人数ぶん（5〜10台）準備。色/IDを割り振る（[03](03-colab-backend.md) の早見表）。
5. ngrok authtoken / Colab シークレットを各アカウントに設定。

---

## 当日の流れ

### ① Colab を起動（開始30〜45分前）

- 必要台数（例: 10人なら 5〜10台）の Colab を起動。1台あたり `CAPACITY=2`。
- 公開トンネルは **`TUNNEL='cloudflare'`** を推奨（無料・鍵不要・複数台OK・警告ページ無し）。
- 各 notebook のセルを上から実行 → 「✅ 準備完了」が出ればOK。
- **Google Sheets を開いて確認**: 起動した台数ぶん行が増え、`lastSeen` が更新され続けていること。

> **トンネルの選択（重要）**: 5〜10台を同時公開するため、本番は **Cloudflare Quick Tunnel** を使います。
> ngrok 無料は「1アカウント＝同時1トンネル」のため複数台には不向きです（ngrok を使うなら台数ぶん別アカウント or 有料）。
> 詳細は [03-colab-backend.md](03-colab-backend.md#トンネルの選び方cloudflare--ngrok)。

### ② iPad を配る

- 各 iPad の Safari で固定URLを開く（事前にブックマーク/ホーム追加が楽）。
- 画面上部に **「あなたは ○サーバー です」** と色が出ることを確認。
- 同じ色が複数iPadに出てもOK（capacity 内なら共有）。

### ③ 制作（子どもの導線）

1. えをえらぶ（20枚から4枚）
2. セリフを書く（4コマ）
3. 声をろくおんする → きいてみる
4. もじにする（文字起こし→必要なら直す）
5. AIで声を作る
6. 4コマげきじょうを見る（▶ さいしょから さいせい）

各ステップは下部の「つぎへ／もどる」で進みます。専門用語は出しません。

### ④ 片付け

- 必要なら各 Colab で `/cleanup` を叩く、または notebook を停止。
- iPad の localStorage をリセットしたい場合は `/admin` →「接続先リセット」。

---

## 混雑・遅延への対処

- AI音声生成は 1台につき**1件ずつ順番**に処理されます（`asyncio.Lock`）。
  同じ色の2人が同時に「声を作る」を押すと、片方は数秒待ちになります（正常動作）。
- 待ちが長い／生成が重い場合は、空いている別Colabへ再割り当て（`/admin` → 再割り当て）。

---

## 管理画面（`/admin`）でできること

子ども画面の**右上の小さな ⚙** から入れます（子どもが触りにくい導線）。

- localStorage の保存内容の確認
- 現在の接続先（serverId / color / apiUrl）
- `/health` 接続確認
- 接続先リセット / 再割り当て
- モード切替（AI / 自分で録音 / ブラウザ読み上げ）
- 緊急時の GAS URL 上書き（この端末のみ）
