# 06. Colab が落ちた時のフォールバック手順

Colab は時間制限や混雑で落ちることがあります。子どもの体験を止めないための多段の保険を用意しています。

---

## 段階1: 自動再割り当て（基本）

保存済みサーバーの `/health` が失敗すると、アプリは自動で動きます。

1. localStorage の接続先に `/health` → 失敗を検知。
2. GAS から一覧を取り直し、**直前のサーバーを除外**して別の生きているColabへ再割り当て。
3. 新しい接続先を localStorage に保存。画面上部の色が変わる。

子ども側の操作は基本不要です。手動でやり直すには `/admin` →「再割り当て」。

接続失敗時は画面に次のボタンが出ます:
- **べつの サーバーに つなぐ**（再割り当て）
- **自分で ろくおんモード**（段階2）
- **読み上げモード**（段階3）

---

## 段階2: 自分で録音モード（`self-record`）

AI音声生成が使えない時に、**子ども自身の声で各コマを録音**して作品を完成させます。

- ネット/サーバー不要（録音はブラウザ内で完結）。
- ステップ: えをえらぶ → セリフ → **コマごとに録音** → 4コマげきじょう。
- 各コマに「● ろくおん／■ ストップ」とファイル選択（保険）があります。
- 切り替え: `/admin` →「自分で録音モード」、または接続失敗バナーのボタン。

---

## 段階3: ブラウザ読み上げモード（`browser-tts`）

録音すらできない / 完全オフラインに近い時の最終手段。

- `speechSynthesis`（端末標準の読み上げ音声）でセリフを読み上げます。
- ステップ: えをえらぶ → セリフ → 4コマげきじょう（録音・生成なし）。
- 4コマ劇場プレイヤーが各コマのセリフを順番に読み上げます。
- 切り替え: `/admin` →「ブラウザ読み上げモード」、または接続失敗バナーのボタン。

> iPad/Safari は日本語の読み上げ音声が入っていれば日本語で読み上げます。
> 端末の音量・サイレントスイッチを確認してください。

---

## モードの優先順位（運用のおすすめ）

```
AIで声を作る（通常）
   └─ 失敗 → 別Colabへ再割り当て（段階1）
        └─ それでもダメ → 自分で録音モード（段階2）
             └─ 録音も不可 → ブラウザ読み上げモード（段階3）
```

全モードで最後の「4コマげきじょう」プレイヤーは共通で、1コマずつ順番に再生できます。

---

## サーバー側が落ちたときの復旧

- Colab セッションが切れたら、notebook のセルを再実行すれば再登録されます（同じ `SERVER_ID` でOK）。
  `register` で `apiUrl` と `lastSeen` が更新され、`assignedCount` は 0 にリセットされます。
- もう使わない台は `/admin` 不要、GAS に `?action=disable&serverId=...` を投げるか、
  Sheets の `enabled` を `false` にすれば割り当て候補から外れます。
- `lastSeen` が `VITE_SERVER_FRESH_SECONDS`（既定120秒）より古い台は自動的に候補外になります。
